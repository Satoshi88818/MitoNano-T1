particle_detect — A Real, Working 3D Particle Detector (+ Full Detect→Track Pipeline)

This document contains a complete, compiling, tested Rust crate implementing MitoNano's V7 detect_particles_3d stub — local-maxima detection, non-maximum suppression, and sub-voxel Gaussian centroid refinement over a 3D intensity field — left as unimplemented!() in the source material. All code below was actually compiled and tested with rustc 1.75.0 / cargo 1.75.0 before being included here.

This document also includes something beyond the stub itself: a Cargo workspace combining this crate with the imm_tracker crate built previously, plus an end-to-end integration test (tests/pipeline.rs) that generates a synthetic "movie" of 3D volumes, detects a moving, maneuvering particle frame by frame with zero knowledge of its true position, and feeds those detections into the IMM tracker — a real, working detect→track pipeline, not two separately-validated pieces that have never been run together.

The detection algorithm

Local-maxima detection (find_local_maxima): every interior voxel whose intensity exceeds a noise floor and is greater than or equal to all 26 of its neighbors.

Non-maximum suppression (non_maximum_suppression): candidates processed brightest-first; a candidate is kept only if it's farther than min_separation from every already-kept candidate — collapses the multiple nearby local maxima that noisy real data produces around a single physical particle down to one detection.

Sub-voxel refinement (refine_subvoxel_position): the standard three-point Gaussian centroid estimator, applied independently along each axis: delta = 0.5 * (ln(I(-1)) - ln(I(+1))) / (ln(I(-1)) - 2*ln(I(0)) + ln(I(+1))) This is the same estimator used across particle-tracking, PIV (particle image velocimetry), and single-molecule localization microscopy for exactly this purpose — not something invented for this crate.

This is the synthetic /// "ground truth particle" used to validate detection accuracy — it's /// deliberately the same functional form the sub-voxel refinement in /// `detect.rs` assumes when it fits a 1D Gaussian along each axis, since /// that assumption is what real fluorescence-microscopy point sources /// approximate reasonably well (an Airy/diffraction-limited spot is /// commonly modeled as an approximately Gaussian PSF for this purpose). pub fn add_gaussian_blob( volume: &mut Volume, center: (f64, f64, f64), amplitude: f64, sigma_voxels: f64, ) { let (cx, cy, cz) = center; for dz in -radius..=radius { for dy in -radius..=radius { for dx in -radius..=radius { let x = cx_i + dx; let y = cy_i + dy; let z = cz_i + dz; if !volume.in_bounds(x, y, z) { continue; } let rx = x as f64 - cx; let ry = y as f64 - cy; let rz = z as f64 - cz; let r_sq = rx * rx + ry * ry + rz * rz; let value = amplitude * (-r_sq / two_sigma_sq).exp(); volume.add(x as usize, y as usize, z as usize, value); } } } } #[cfg(test)] mod tests { use super::*; particle_detect/src/detect.rs

//! Particle detection: finds local-intensity peaks in a 3D volume above a //! noise floor, suppresses duplicate detections of the same physical //! particle, and refines each surviving peak's position to sub-voxel //! accuracy via a three-point Gaussian centroid estimator. //! //! This is what MitoNano's V7 `detect_particles_3d` stub named //! ("local-maxima detection + sub-voxel centroid refinement over the 3D //! intensity field") but left as `unimplemented!()`. It's also the piece //! that turns the `imm_tracker` crate from "a tracker that accepts //! measurements from somewhere" into part of an actual detect-then-track //! pipeline — see `tests/pipeline.rs` in this workspace for that full //! chain running end to end. use nalgebra::Point3; use crate::volume::Volume; /// One detected particle: its refined sub-voxel position and the raw /// intensity at its integer-voxel peak (useful for confidence filtering, /// downstream weighting, etc., though not used further by this crate). #[derive(Clone, Copy, Debug, PartialEq)] pub struct DetectedParticle3D { pub position: Point3<f64>, pub peak_intensity: f64, } /// Detects particles in `volume`: local maxima above `noise_floor`, /// deduplicated so detections of the same physical particle (multiple /// nearby voxels all locally brighter than their immediate neighbors, /// which real noisy data produces routinely) collapse to one, each /// refined to sub-voxel accuracy. /// /// `min_separation_voxels` sets how close two accepted detections are /// allowed to be; anything closer is treated as the same particle and the /// dimmer candidate is dropped. pub fn detect_particles_3d( volume: &Volume, noise_floor: f64, min_separation_voxels: f64, ) -> Vec<DetectedParticle3D> { let candidates = find_local_maxima(volume, noise_floor); let suppressed = non_maximum_suppression(candidates, min_separation_voxels); suppressed .into_iter() .filter_map(|(x, y, z, intensity)| refine_subvoxel_position(volume, x, y, z, intensity)) .collect() } /// Step 1: every interior voxel whose intensity exceeds `noise_floor` and /// is greater than or equal to all 26 of its neighbors. Border voxels are /// excluded (both to avoid an out-of-bounds neighbor check and because /// sub-voxel refinement needs a neighbor on each side anyway). fn find_local_maxima(volume: &Volume, noise_floor: f64) -> Vec<(usize, usize, usize, f64)> { let mut maxima = Vec::new(); if volume.nx < 3 || volume.ny < 3 || volume.nz < 3 { return maxima; } for z in 1..volume.nz - 1 { for y in 1..volume.ny - 1 { for x in 1..volume.nx - 1 { let center = volume.get(x, y, z); if center <= noise_floor { continue; } let mut is_max = true; 'neighbors: for dz in -1..=1i32 { for dy in -1..=1i32 { for dx in -1..=1i32 { if dx == 0 && dy == 0 && dz == 0 { continue; } let nx = (x as i32 + dx) as usize; let ny = (y as i32 + dy) as usize; let nz = (z as i32 + dz) as usize; if volume.get(nx, ny, nz) > center { is_max = false; break 'neighbors; } } } } if is_max { maxima.push((x, y, z, center)); } } } } maxima } /// Step 2: greedy non-maximum suppression. Candidates are processed /// brightest-first; a candidate is kept only if it's farther than /// `min_separation` from every already-kept candidate. This is the same /// greedy strategy commonly used for detection deduplication generally /// (e.g. bounding-box NMS in object detection), applied to point /// detections instead of boxes. fn non_maximum_suppression( mut candidates: Vec<(usize, usize, usize, f64)>, min_separation: f64, ) -> Vec<(usize, usize, usize, f64)> { candidates.sort_by(|a, b| b.3.partial_cmp(&a.3).unwrap_or(std::cmp::Ordering::Equal)); let min_sep_sq = min_separation * min_separation; let mut kept: Vec<(usize, usize, usize, f64)> = Vec::new(); for candidate in candidates { let (cx, cy, cz, _) = candidate; let far_enough = kept.iter().all(|&(kx, ky, kz, _)| { let dx = cx as f64 - kx as f64; let dy = cy as f64 - ky as f64; let dz = cz as f64 - kz as f64; dx * dx + dy * dy + dz * dz >= min_sep_sq }); if far_enough { kept.push(candidate); } } kept } /// Step 3: three-point Gaussian sub-voxel centroid estimator, applied /// independently along each axis. For an intensity profile that's /// genuinely Gaussian along an axis, `I(u) = A * exp(-(u-u0)^2 / (2*sigma^2))`, /// the sub-voxel peak offset from the integer maximum can be recovered /// exactly (in the noiseless case) from just the three samples /// `I(-1), I(0), I(+1)` around it: /// /// ```text /// delta = 0.5 * (ln(I(-1)) - ln(I(+1))) / (ln(I(-1)) - 2*ln(I(0)) + ln(I(+1))) /// ``` /// /// This is the standard estimator used across particle-tracking, /// PIV (particle image velocimetry), and single-molecule localization /// microscopy for exactly this purpose. Falls back to the integer /// position (no sub-voxel refinement) if any of the three samples is /// non-positive, since the logarithm is undefined there — a legitimate, /// visible degradation rather than a silent NaN. fn refine_subvoxel_position( volume: &Volume, x: usize, y: usize, z: usize, peak_intensity: f64, ) -> Option<DetectedParticle3D> { let axis_offset = |i_minus: f64, i_center: f64, i_plus: f64| -> f64 { if i_minus <= 0.0 || i_center <= 0.0 || i_plus <= 0.0 { return 0.0; } let ln_minus = i_minus.ln(); let ln_center = i_center.ln(); let ln_plus = i_plus.ln(); let denom = ln_minus - 2.0 * ln_center + ln_plus; if denom.abs() < 1e-12 { // Degenerate (near-flat top, e.g. a saturated/clipped peak) — // no reliable curvature to fit against; fall back to zero // offset rather than dividing by ~0. return 0.0; } 0.5 * (ln_minus - ln_plus) / denom }; let dx = axis_offset(volume.get(x - 1, y, z), peak_intensity, volume.get(x + 1, y, z)); let dy = axis_offset(volume.get(x, y - 1, z), peak_intensity, volume.get(x, y + 1, z)); let dz = axis_offset(volume.get(x, y, z - 1), peak_intensity, volume.get(x, y, z + 1)); // Guard against a pathological offset larger than half a voxel, which // would indicate the three-point fit isn't trustworthy here (e.g. two // overlapping peaks distorting the local profile) rather than a // genuine sub-voxel correction. let clamp = |d: f64| d.clamp(-0.5, 0.5); Some(DetectedParticle3D { position: Point3::new(x as f64 + clamp(dx), y as f64 + clamp(dy), z as f64 + clamp(dz)), peak_intensity, }) } #[cfg(test)] mod tests { use super::*; use crate::volume::add_gaussian_blob; #[test] fn three_point_estimator_recovers_known_subvoxel_offset() { // Directly synthesize three samples of a 1D Gaussian with a known // sub-voxel peak offset and confirm the closed-form estimator // recovers it, independent of the 3D detection pipeline. let sigma: f64 = 2.0; let true_offset: f64 = 0.3; // peak sits 0.3 voxels to the right of sample 0 let gaussian = |u: f64| (-((u - true_offset).powi(2)) / (2.0 * sigma * sigma)).exp(); let i_minus = gaussian(-1.0); let i_center = gaussian(0.0); let i_plus = gaussian(1.0); let ln_minus = i_minus.ln(); let ln_center = i_center.ln(); let ln_plus = i_plus.ln(); let denom = ln_minus - 2.0 * ln_center + ln_plus; let recovered_offset = 0.5 * (ln_minus - ln_plus) / denom; assert!( (recovered_offset - true_offset).abs() < 1e-9, "recovered {recovered_offset}, expected {true_offset}" ); } #[test] fn detects_single_isolated_blob_at_known_subvoxel_position() { let mut v = crate::volume::Volume::zeros(31, 31, 31); particle_detect/tests/pipeline.rs

//! The payoff of building `particle_detect` alongside `imm_tracker`: a //! genuine end-to-end pipeline. A synthetic "movie" of 3D intensity //! volumes is generated frame by frame for a single moving, maneuvering //! particle plus background noise. Each frame is run through //! `detect_particles_3d` — with no knowledge of where the particle //! "should" be — and the resulting detection is fed as a measurement into //! an `imm_tracker::ImmTracker`. No part of this test hands the tracker a //! true position directly; every measurement genuinely comes from pixels. //! //! This is what "detect → track" means in the MitoNano write-up's own //! architecture, and it's the reason `particle_detect` was picked as the //! next piece to build after `imm_tracker`: on its own, an IMM tracker //! needs measurements from *somewhere*; on its own, a detector produces //! per-frame points with no memory across frames. Together, they're a //! minimal working tracking system. use imm_tracker::imm::{ImmParams, ImmTracker}; use imm_tracker::kalman::{MeasurementVector, StateCovariance, StateVector}; use particle_detect::detect::detect_particles_3d; use particle_detect::volume::{add_gaussian_blob, Volume}; /// Minimal deterministic PRNG for reproducible background noise, same /// approach used in `imm_tracker`'s own integration test — avoids an /// external `rand` dependency purely for test determinism. struct Lcg { state: u64, } impl Lcg { fn new(seed: u64) -> Self { Lcg { state: seed } } fn next_unit(&mut self) -> f64 { self.state = self.state.wrapping_mul(6364136223846793005).wrapping_add(1442695040888963407); (self.state >> 40) as f64 / (1u64 << 24) as f64 // in [0, 1) } } /// True (unobserved) particle trajectory: constant velocity, then a /// sustained acceleration maneuver, then constant velocity again — same /// three-phase shape as `imm_tracker`'s own maneuvering-target test, so /// the two validate the same kind of scenario at different levels of the /// pipeline (that test: noisy positions straight to the tracker; this /// test: noisy pixels through a detector first). fn true_position(frame: usize, v0: f64, cv1_frames: usize, ca_frames: usize, accel: f64) -> (f64, f64, f64) { let t = frame as f64; let t1 = cv1_frames as f64; let t2 = (cv1_frames + ca_frames) as f64; let x = if t <= t1 { v0 * t } else if t <= t2 { let d = t - t1; v0 * t1 + v0 * d + 0.5 * accel * d * d } else { let x2 = v0 * t1 + v0 * (t2 - t1) + 0.5 * accel * (t2 - t1) * (t2 - t1); let v2 = v0 + accel * (t2 - t1); x2 + v2 * (t - t2) }; (5.0 + x, 20.0, 20.0) } /// Renders one frame: a Gaussian blob at the true particle position plus /// uniform background noise, comfortably below the detector's noise /// floor everywhere except right at the particle. fn render_frame(nx: usize, ny: usize, nz: usize, particle_pos: (f64, f64, f64), rng: &mut Lcg) -> Volume { let mut volume = Volume::zeros(nx, ny, nz); for z in 0..nz { for y in 0..ny { for x in 0..nx { volume.set(x, y, z, rng.next_unit() * 8.0); // background in [0, 8) } } } add_gaussian_blob(&mut volume, particle_pos, 600.0, 1.8); volume } #[test] fn detects_and_tracks_a_maneuvering_particle_across_a_synthetic_movie() { let nx = 75; let ny = 41; let nz = 41; let v0 = 0.5; let cv1_frames = 15; let ca_frames = 8; let cv2_frames = 10; let total_frames = cv1_frames + ca_frames + cv2_frames; let accel = 0.3; let noise_floor = 60.0; let min_separation = 3.0; let mut rng = Lcg::new(0xBEEF_CAFE); let initial_true = true_position(0, v0, cv1_frames, ca_frames, accel); let initial_state = { let mut s = StateVector::zeros(); s[0] = initial_true.0; s[1] = initial_true.1; s[2] = initial_true.2; s }; let initial_covariance = StateCovariance::identity() * 4.0; let params = ImmParams::defaults(0.15); let mut tracker = ImmTracker::new(initial_state, initial_covariance, [0.9, 0.1], params); let mut squared_errors = Vec::with_capacity(total_frames); let mut detections_found = 0usize; for frame in 1..=total_frames { let true_pos = true_position(frame, v0, cv1_frames, ca_frames, accel); let volume = render_frame(nx, ny, nz, true_pos, &mut rng); let detections = detect_particles_3d(&volume, noise_floor, min_separation); assert_eq!( detections.len(), 1, "frame {frame}: expected exactly one detection (the real particle), got {}", detections.len() ); detections_found += 1; let d = detections[0]; let z = MeasurementVector::new(d.position.x, d.position.y, d.position.z); let report = tracker.step(1.0, Some(z)); let est = report.combined_state; let err_sq = (est[0] - true_pos.0).powi(2) + (est[1] - true_pos.1).powi(2) + (est[2] - true_pos.2).powi(2); squared_errors.push(err_sq); } assert_eq!(detections_found, total_frames, "every frame in the synthetic movie should yield a detection"); let rms_error = (squared_errors.iter().sum::<f64>() / squared_errors.len() as f64).sqrt(); println!("end-to-end pipeline RMS tracking error = {rms_error:.4} voxels over {total_frames} frames"); // The detector alone is accurate to well under 0.1 voxel per frame // (see particle_detect's own tests); this bound is looser to also // account for the IMM tracker's own filtering dynamics (one-step lag // through the maneuver, etc.), not just detector noise. assert!( rms_error < 0.5, "end-to-end RMS tracking error ({rms_error:.4}) should stay well under half a voxel across the whole synthetic movie" ); }

A real bug this process found and fixed

The first version of the pipeline test failed: frame 23 produced zero detections. Tracing it down: the synthetic trajectory's acceleration maneuver pushed the particle's true position (x ≈ 66.4) past the edge of the 60-voxel-wide test volume — the Gaussian blob was being rendered partially or fully outside the array bounds, so there was nothing there for the detector to find. This wasn't a detection-algorithm bug; it was a test-scenario sizing bug (not checking that a physically reasonable trajectory actually stays inside the volume being simulated). Fixed by recomputing the trajectory's extent analytically and choosing gentler motion parameters (v0, accel, frame counts) with headroom, and widening the volume to 75 voxels along the axis of motion. Left in as documentation because it's a genuinely easy mistake to make when wiring synthetic test data together, and catching it required actually running the pipeline against concrete numbers — exactly the kind of thing a prose description of "generate a synthetic movie" would never surface.

How to build and run this yourself

mkdir pipeline && cd pipeline # create Cargo.toml (workspace), particle_detect/, and imm_tracker/ # with the contents above (imm_tracker unchanged from the prior deliverable) cargo test --workspace 

Requires only nalgebra from crates.io for both crates — no GPU, no ML framework, no external dataset, no image-format dependency (volumes are plain in-memory Vec<f64>). Builds and passes on stock rustc 1.75.0.

Scope, stated plainly

This detects particles in a single 3D intensity volume (one time step) and demonstrates feeding a sequence of such detections into a tracker. It does not implement: multi-target data association (matching each of several simultaneous detections to the correct existing track — this demo has exactly one particle, so association is trivial); track initiation/deletion logic for particles entering or leaving the field of view; real image I/O (reading actual microscopy file formats — volumes here are synthetic, in-memory arrays); or any GPU acceleration of the detection loop (it's a plain CPU triple-nested loop, fine at this test scale but not optimized for large real volumes). Those are real, separate pieces of a production tracking system, not implied by anything above.
