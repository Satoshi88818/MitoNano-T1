Part 1: Why "orchestration" means something different here

The original MitoNano document series used "orchestration" for federated multi-institution trust ledgers and quorum-governed cost routing — infrastructure for a deployment scale and governance that are not currently needed in accordance with current technological limitations. At this project's actual scale (a single in-memory process, tested up to 800 agents so far), organizing agents concretely means one real, well-understood thing: a spatial index, so "which agents are near this point?" is a fast bucketed lookup instead of an O(n²) scan over every agent. That's what orchestration.rs does, and deliberately nothing else — no governance, no distributed trust, no multi-node anything, because none of that has a demonstrated need yet.

SpatialIndex — what it does

A standard uniform grid / spatial hash:

rebuild(agents) — buckets every agent index into a 3D grid cell based on current position. Called once per tick, after motion.

bucket_of(position) — clamped cell lookup (same "clamp rather than panic" policy as ConcentrationField).

neighbors_within(agents, query, radius) — broad-phase (only searches cells that could contain a match, computed from ceil(radius/cell_width) per axis, so it's correct for any radius, not just sub-cell ones) followed by an exact narrow-phase distance check.

This is textbook computational geometry, which is exactly why it has an unambiguous correctness criterion: brute-force all-pairs search must find the identical set of neighbors, just slower.

The correctness tests

rebuild_indexes_every_agent_exactly_once — 500 random agents, checked via a HashSet that every agent appears in exactly one bucket: none lost, none double-counted.

bucket_assignment_matches_direct_computation — cell assignment checked against an independently-written formula at corner, edge, and exact-grid-line positions.

neighbors_within_matches_brute_force_search — 800 random agents, 30 random query/radius combinations, indexed result compared as a HashSet against brute-force O(n) search. Must match exactly, not just overlap.

neighbors_within_matches_brute_force_for_large_radius — same check with radius larger than a single cell, exercising the multi-layer search path.

rebuild_after_movement_updates_membership — an agent that moves to a new region is confirmed gone from its old bucket and present in the new one after rebuild.

query_with_no_nearby_agents_returns_empty — edge case: empty result near a domain boundary, no panic on out-of-range layer search.

45 total random query/radius trials across the two brute-force comparison tests, all exact matches.

Part 2: Finishing the feedback loop (field → agent uptake)

Last time, coupling was one-directional (agent → field, via release). This pass adds the missing direction: agents can now withdraw mass from the field back into their own payload — ConcentrationField::withdraw is capped at whatever mass is actually present in a cell, so it can never be driven negative, and returns the actual amount transferred so callers credit exactly that (not the requested amount) to keep the system exactly conservative.

CoupledEngine::run_with_feedback ties it together per tick: agent moves → senses local concentration → withdraws an uptake amount (k_uptake × local_concentration × dt, capped) → gains it into payload → undergoes exact decay split (release deposited back into the field, elimination lost) → field diffuses.

The invariant tests

full_feedback_loop_exactly_conserves_mass_with_zero_elimination — with k_elim=0, combined mass (agents + field) stays exactly constant across 500 ticks of active two-way exchange (some agents releasing, others only gaining via uptake), to 1e-9 relative tolerance at every tick.

full_feedback_loop_matches_exact_bookkeeping_identity_under_full_complexity — with motion, diffusion, release, uptake, and elimination all active simultaneously, there's no simple closed form anymore (uptake depends on the field's diffusion history, which depends on every agent's trajectory) — but the discrete identity combined_mass(t) == C0 - cumulative_eliminated(t) must still hold exactly, because release/deposit and uptake/withdraw are each self-canceling transfers and elimination is the only real sink. Checked to 1e-9 at every tick over 700 steps, 40 agents.

full_feedback_loop_never_produces_negative_concentration — 100 agents crowded into a small region to force heavy same-cell uptake contention, checked at every tick that no sampled concentration goes negative.

Each tick: every agent /// moves and releases payload (via `step_coupled`), the released /// amount is deposited into the field at the agent's *new* (post- /// motion) position, and then the field diffuses one step.
