Output Tooling: Actually Seeing What This Simulates

Up to now, every run was inspected only through aggregate scalars in test assertions. This adds real export: agent trajectories, full field snapshots, and per-tick stats, all as CSV.

Status: compiles clean, zero warnings. 7 new tests, all passing. Full crate: 57/57 tests passing in ~2s. Also run end-to-end against a real 40-agent, 300-tick UnifiedEngine simulation (not just unit-tested in isolation) — see the demo run below, including a real bug it caught in the demo script itself.

Design: CSV, deliberately boring

No plotting library, no new dependency. CSV is the maximally portable choice — importable into a spreadsheet, pandas, R, or gnuplot without this crate (whose job is the simulation, not visualization) taking on a graphics dependency.

Three writers:

write_agent_snapshots_csv — tick,agent_index,x,y,z,payload, from a Vec<AgentSnapshot> built by snapshot_agents(tick, agents). Call periodically during a run (not every tick) to keep file size sane.

write_field_csv — tick,ix,iy,iz,x,y,z,concentration, one row per grid cell. Required exposing two small new accessors on ConcentrationField (cell_value and making cell_center public) — previously the field's internals were only reachable through sample() (position-based) and the private test helper.

write_unified_tick_stats_csv — the aggregate conservation stats already being computed, just written out instead of only asserted on.

The tests: reconstruct the invariant from the file, not just check the text looks right

This module's correctness bar is different from the rest of the crate — there's no analytical solution for a CSV writer. So the tests do the crate's usual thing in a new way: parse the written bytes back and reconstruct a physical invariant purely from the text, then compare to the source data's own accounting.

agent_snapshot_csv_round_trips_total_payload_exactly — sums the payload column parsed back from 25 agents' CSV rows, checks it matches the original sum to 1e-9. Also spot-checks one specific agent's position round-trips correctly.

field_csv_round_trips_total_mass_exactly — sums concentration × cell_volume across every one of 1,331 exported cell rows (11³ grid), checks it reconstructs field.total_mass() to 1e-6.

field_csv_coordinates_match_cell_center_exactly — the exported (x,y,z) for a specific cell must match ConcentrationField::cell_center() directly, not some independently (and possibly inconsistently) computed coordinate.

Plus row-count, header-correctness, and empty-input edge case tests.

Running it against a real simulation (not just unit tests)

examples/export_simulation.rs runs a 40-agent UnifiedEngine for 300 ticks, exporting field snapshots at the start and end, agent trajectories every 10 ticks, and per-tick stats throughout.

This caught a real bug in the demo script, worth including rather than hiding: UnifiedEngine::run(1, ...) resets its internal tick counter to 0..=1 on every call (it's not a running total), so calling it in a loop and blindly recording stats[1].tick produced a CSV where every row claimed tick=1. Fixed by overriding the stat's tick field with the actual loop counter before recording. This is exactly the kind of bug that only shows up when you actually run the thing end-to-end — a unit test on the writer alone wouldn't have caught a caller-side misunderstanding of the engine's API.

After the fix, the output makes physical sense:

Field mass climbs from 0 (empty at tick 0) to ~54 as agents release payload; total combined mass declines smoothly and only via cumulative_eliminated growing — exactly the physics UnifiedEngine's own tests already proved, now visible in an actual file instead of only asserted on.

//! 3D voxel concentration field, evolved by explicit finite-difference //! diffusion (FTCS scheme) with zero-flux (Neumann) boundaries. //! //! This is what makes agents able to interact indirectly (deposit payload //! into the field, sense local concentration) instead of moving as fully //! independent particles. Deliberately kept to the standard, well-studied //! explicit scheme rather than anything fancier (implicit/Crank-Nicolson, //! adaptive mesh, etc.) since correctness here is checked against the //! textbook analytical solution for diffusion from a point source, and //! that comparison is only clean for the simplest possible discretization. use crate::environment::Bounds; pub struct ConcentrationField { pub bounds: Bounds, pub nx: usize, pub ny: usize, pub nz: usize, dx: f64, dy: f64, dz: f64, /// Flattened row-major storage: index = ix + nx*(iy + ny*iz). cells: Vec<f64>, } impl ConcentrationField { pub fn new(bounds: Bounds, nx: usize, ny: usize, nz: usize) -> Self { assert!(nx >= 3 && ny >= 3 && nz >= 3, "grid needs at least 3 cells per axis for a Laplacian stencil"); let dx = (bounds.max[0] - bounds.min[0]) / nx as f64; let dy = (bounds.max[1] - bounds.min[1]) / ny as f64; let dz = (bounds.max[2] - bounds.min[2]) / nz as f64; Self { bounds, nx, ny, nz, dx, dy, dz, cells: vec![0.0; nx * ny * nz] } } #[inline] fn idx(&self, ix: usize, iy: usize, iz: usize) -> usize { ix + self.nx * (iy + self.ny * iz) } pub fn cell_volume(&self) -> f64 { self.dx * self.dy * self.dz } /// Concentration in a specific cell, addressed by grid index rather /// than world position. Used by output tooling to export the full /// field grid (`sample` is the position-based equivalent used by the /// simulation itself). pub fn cell_value(&self, ix: usize, iy: usize, iz: usize) -> f64 { self.cells[self.idx(ix, iy, iz)] } /// Total mass currently in the field (concentration integrated over /// volume). Used as the primary conservation invariant: with zero-flux /// boundaries this must stay constant across `step` calls to floating /// point precision, regardless of grid resolution or diffusion /// coefficient (as long as the stability condition holds). pub fn total_mass(&self) -> f64 { self.cells.iter().sum::<f64>() * self.cell_volume() } /// Maps a world-space position to the voxel that contains it, clamped /// to the grid (so a position exactly on or outside the boundary still /// resolves to a valid cell rather than panicking). fn position_to_index(&self, p: [f64; 3]) -> (usize, usize, usize) { let clamp_axis = |v: f64, lo: f64, d: f64, n: usize| -> usize { let raw = ((v - lo) / d).floor(); raw.max(0.0).min((n - 1) as f64) as usize }; ( clamp_axis(p[0], self.bounds.min[0], self.dx, self.nx), clamp_axis(p[1], self.bounds.min[1], self.dy, self.ny), clamp_axis(p[2], self.bounds.min[2], self.dz, self.nz), ) } /// Deposits `amount` of mass into the voxel containing `position`, as /// a concentration increment (amount / cell_volume). This is the /// standard finite-volume way to inject a point source: it exactly /// conserves total mass by construction (mass added = amount, always). pub fn deposit(&mut self, position: [f64; 3], amount: f64) { let (ix, iy, iz) = self.position_to_index(position); let i = self.idx(ix, iy, iz); self.cells[i] += amount / self.cell_volume(); } /// Samples the concentration at the voxel containing `position` /// (nearest-cell sampling; no interpolation). pub fn sample(&self, position: [f64; 3]) -> f64 { let (ix, iy, iz) = self.position_to_index(position); self.cells[self.idx(ix, iy, iz)] } /// Withdraws up to `amount` of mass from the voxel containing /// `position`, returning the amount actually removed. This is the /// field-to-agent counterpart of `deposit`: capped at the cell's /// currently available mass so the field can never be driven negative, /// regardless of how large a withdrawal is requested (e.g. several /// agents in the same cell in one tick, or a caller-computed uptake /// rate that outpaces what's locally present). The returned value is /// the authoritative amount transferred -- callers should credit the /// recipient with exactly this, not with the originally requested /// amount, to keep total system mass exactly conserved. pub fn withdraw(&mut self, position: [f64; 3], amount: f64) -> f64 { if amount <= 0.0 { return 0.0; } let (ix, iy, iz) = self.position_to_index(position); let i = self.idx(ix, iy, iz); let available_mass = self.cells[i] * self.cell_volume(); let actual = amount.min(available_mass).max(0.0); self.cells[i] -= actual / self.cell_volume(); actual } /// The largest `dt` for which the explicit FTCS scheme is guaranteed /// stable and positivity-preserving for this grid and diffusion /// coefficient: dt <= 1 / (2*D*(1/dx^2 + 1/dy^2 + 1/dz^2)). /// Derived from requiring the center-cell update coefficient to stay /// non-negative; violating it can produce oscillation or negative /// concentrations. pub fn max_stable_dt(&self, diffusion_coeff: f64) -> f64 { let sum_inv_sq = 1.0 / (self.dx * self.dx) + 1.0 / (self.dy * self.dy) + 1.0 / (self.dz * self.dz); 1.0 / (2.0 * diffusion_coeff * sum_inv_sq) } /// Advances the field by one explicit FTCS diffusion step. /// Zero-flux boundaries are implemented by clamping the neighbor index /// at each face to the boundary cell itself (a standard finite-volume /// no-flux treatment: the flux term at a boundary face becomes /// (C_boundary - C_boundary) = 0 by construction, so no mass crosses /// the domain edge and `total_mass` is exactly preserved). /// /// Panics if `dt` exceeds `max_stable_dt` for the given diffusion /// coefficient, since the update would otherwise be numerically /// invalid (not merely "less accurate"). pub fn step(&mut self, dt: f64, diffusion_coeff: f64) { let max_dt = self.max_stable_dt(diffusion_coeff); let inv_dy2 = 1.0 / (self.dy * self.dy); let inv_dz2 = 1.0 / (self.dz * self.dz); for iz in 0..self.nz { let iz_lo = iz.saturating_sub(1); let iz_hi = (iz + 1).min(self.nz - 1); for iy in 0..self.ny { let iy_lo = iy.saturating_sub(1); next[self.idx(ix, iy, iz)] = c + dt * diffusion_coeff * laplacian; } } } self.cells = next; } /// Cell-center world-space coordinate for a given voxel index. Used by /// tests for the analytical-solution comparison, and by output /// tooling to attach real-world coordinates to each exported cell. pub fn cell_center(&self, ix: usize, iy: usize, iz: usize) -> [f64; 3] { [ self.bounds.min[0] + (ix as f64 + 0.5) * self.dx, self.bounds.min[1] + (iy as f64 + 0.5) * self.dy, self.bounds.min[2] + (iz as f64 + 0.5) * self.dz, ] } } #[cfg(test)] mod tests { use super::*; fn cube(half_width: f64, n: usize) -> ConcentrationField { let bounds = Bounds::new([-half_width, -half_width, -half_width], [half_width, half_width, half_width]); ConcentrationField::new(bounds, n, n, n) } /// With zero-flux boundaries, total mass must be exactly conserved /// across any number of steps, independent of grid resolution or how /// close the plume gets to the boundary.
